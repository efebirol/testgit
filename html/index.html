  • open the "git bash"

Initializing, adding, committing, and status

=============================================

  • "git init" initialisiert das Projekt
    • erstellt eine .git Datei

  • füge die Datei hinzu (erstmal nur die Staging Umgebung, also ein Schritt vor der remote Repository)
    • "git add " Info: "git add --all" pusht alles in das Staging (neue Dateien, Löschungen, Änderungen etc.)

  • prüfe den Status
  • "git status"
    • zeigt an welche Datei in der Stage Phase sind (also vor dem commit) und welche nicht

  • Info: Farben beim "git Status" Befehl
    • untracked files = magenta
    • new files = green
    • modified files = blue
    • deleted files = red

  • git commit -m "revision one"
    • Commit Änderung auf die entfernte Repository

  • Tip: Änderung an die Email senden
    • git config --global user.email ""
    • git config --global user.name ""

  • "git log"
    • einsehen was alles committed wurde
